## 생성형 AI (GenAI)란?

대화, 스토리, 이미지, 동영상, 음악 등 새로운 콘텐츠와 아이디어를 생성하는, 대규모 데이터를 기반으로 사전 학습된 초거대 모델로, 일반적으로 **파운데이션 모델 (Foundation Model, FM)** 이라고 합니다.

<!--more-->

### 전통적 기계학습 모델과 파운데이션 모델 비교

| 구분 | 전통적 ML 모델 | 파운데이션 모델 |
|------|---------------|----------------|
| 학습 방식 | 특정 태스크용 학습 | 대규모 범용 사전학습 |
| 적용 범위 | 단일 태스크 | 다양한 downstream 태스크 |
| 데이터 요구량 | 상대적으로 적음 | 대규모 데이터 필요 |
| 재사용성 | 낮음 | 높음 (Fine-tuning 가능) |

**파운데이션 모델(Foundation Model)** 은 대규모 데이터로 사전학습된 AI 모델로, 다양한 downstream 태스크에 적용될 수 있는 기반 모델입니다.

### 생성형 AI 활용 시 도전 과제

- **학습 데이터에 의존적**: 학습 시점의 데이터에 한정됨
- **기한이 지난 정보**: 실시간 정보 반영 불가
- **환각 현상(Hallucination)**: 부정확한 사실을 자신있게 제공
- **맥락 이해 부족**: 컨텍스트 이해 및 추론의 한계

### 파운데이션 모델의 도메인 적응 방법

복잡도와 비용 순서로 나열하면 다음과 같습니다.

1. **컨텍스트 기반 프롬프트 엔지니어링**: 가장 간단하고 비용 효율적
2. **검색 증강 생성 (RAG) 활용**: 외부 지식 베이스 연동
3. **사전 훈련된 파운데이션 모델 파인 튜닝**: 도메인 특화 학습
4. **자체적인 파운데이션 모델 훈련**: 가장 높은 비용과 복잡도

## 검색 증강 생성 (RAG, Retrieval-Augmented Generation)

RAG는 사용자 질문을 지식 베이스에서 검색하여 프롬프트를 증강함으로써, 파운데이션 모델이 생성하는 답변의 정확성과 신뢰성을 높이는 기법입니다.

### RAG 동작 흐름

```
┌─────────────────────────────────────────────────────────────────┐
│  1. 최신/특정 도메인 지식 베이스 생성 (배치성 작업)                    │
│     ↓                                                           │
│  2. 사용자 질문 제출 (실시간)                                        │
│     ↓                                                           │
│  3. 질문 임베딩 → 지식 베이스 질의 → 유사 문서 검색 → 컨텍스트 변환     │
│     ↓                                                           │
│  4. 질문 + 컨텍스트 → 확장된 프롬프트 생성                            │
│     ↓                                                           │
│  5. 신뢰성 높은 답변 생성                                           │
└─────────────────────────────────────────────────────────────────┘
```

> **핵심 포인트**: 지식 베이스를 높은 품질로 구축하는 것이 RAG 성능의 핵심입니다.

### RAG 지식 베이스 구축 프로세스

1. **데이터 수집**: 원시 데이터 (이미지, 문서, 오디오)
2. **전처리**: 데이터 추출, 청크 단위 분할
3. **임베딩 생성**: 인코더를 활용한 벡터 임베딩
4. **저장**: 벡터 데이터베이스에 적재
5. **활용**: 실시간 유사도 검색

## 벡터 데이터베이스

> 실시간 벡터 임베딩을 저장, 변경, 관리하고 고성능 벡터 유사성 검색 알고리즘을 제공하는 특수 목적 데이터베이스

### 주요 특징

- 수백~수천 차원의 벡터 데이터를 효율적으로 저장
- 텍스트, 이미지, 오디오 등 다양한 데이터 타입의 임베딩 처리
- 벡터와 함께 원본 데이터, 태그, 타임스탬프 등 메타데이터 저장
- 필터링과 조건 검색 지원

### 대표적인 벡터 데이터베이스

| 종류 | 특징 |
|------|------|
| Pinecone | 완전 관리형, 서버리스 |
| Weaviate | 오픈소스, GraphQL 지원 |
| Chroma | 경량, 로컬 개발에 적합 |
| pgvector (PostgreSQL) | 기존 PostgreSQL 확장 |
| Amazon OpenSearch | AWS 네이티브 통합 |

### 핵심 개념

#### 벡터 (Vector)

- 크기와 방향을 함께 갖는 양
- N차원의 원소들의 배열 (1×N 행렬)
- 예: `(0.275, 0.827, -0.133, ..., -0.394)`

#### 임베딩 (Embedding)

단어, 문장, 문서, 이미지 등의 데이터를 모델의 인코더를 활용하여 컴퓨터가 이해할 수 있는 형태의 벡터로 변환하는 과정 또는 변환된 벡터입니다.

#### 벡터 간의 유사성 측정

- **코사인 유사도 (Cosine Similarity)**: 벡터 간 각도 기반
- **유클리드 거리 (Euclidean Distance)**: 벡터 간 직선 거리
- **내적 (Dot Product)**: 벡터의 내적값 활용

### 벡터 검색 알고리즘

#### k-NN (k-Nearest Neighbors)

- 정확한 최근접 이웃 검색
- 모든 벡터와 비교하므로 대규모 데이터에서 느림

#### ANN (Approximate Nearest Neighbors)

- 근사 최근접 이웃 검색
- 약간의 정확도를 희생하고 속도 향상
- 대규모 데이터셋에 적합

### ANN 기반 인덱싱 알고리즘

#### IVFFlat (Inverted File with Flat Compression)

- K-means 기반 버킷(클러스터) 구성
- 각 버킷에 속하는 벡터를 리스트로 색인
- 빠른 인덱싱 속도
- 주요 매개변수: `nlist` (버킷 수), `nprobe` (검색할 버킷 수)

#### HNSW (Hierarchical Navigable Small World)

- 그래프 기반 계층적 구조
- 대규모 데이터셋에서 높은 성능과 재현율
- 주요 매개변수: `M` (연결 수), `ef_construction`, `ef_search`

### 벡터 데이터베이스 동작 과정

```
┌──────────────┐    ┌──────────────┐    ┌──────────────┐
│   인덱싱     │ →  │    쿼리      │ →  │   후처리     │
│              │    │              │    │              │
│ 벡터 저장 및 │    │ 쿼리 벡터와  │    │ 메타데이터   │
│ 인덱스 구축  │    │ 유사도 비교  │    │ 필터링/정렬  │
└──────────────┘    └──────────────┘    └──────────────┘
```
